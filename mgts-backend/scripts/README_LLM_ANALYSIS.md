# Анализ страниц с использованием LLM (Perplexity)

Скрипт `analyze-page-with-llm.js` использует Perplexity API для семантического анализа веб-страниц и создания технических заданий (ТЗ) для редизайна.

## Возможности

- 🤖 **Семантический анализ** - понимание смысла и назначения контента
- 📋 **Классификация контента** - автоматическое определение типов секций
- 📝 **Обобщающие описания** - создание кратких описаний сложных блоков
- 🎯 **Структурированный вывод** - JSON формат для дальнейшей обработки

## Использование

### Базовое использование (режим: только скриншот - рекомендуется)

```bash
cd mgts-backend
node scripts/analyze-page-with-llm.js home
```

По умолчанию используется режим анализа **только по скриншоту**, который показал лучшие результаты для семантического анализа.

### С указанием API ключа через переменную окружения

```bash
export PERPLEXITY_API_KEY="pplx-your-key-here"
node scripts/analyze-page-with-llm.js home
```

### Автоматическое чтение ключа из CONTEXT.md

Скрипт автоматически читает API ключ из `docs/project/CONTEXT.md`, если переменная окружения не установлена.

### Режимы анализа

Скрипт поддерживает два режима анализа:

1. **`screenshot`** (по умолчанию, рекомендуется) - анализ только по скриншоту страницы
   - Более точный семантический анализ
   - Понимает визуальную структуру
   - Игнорирует технические детали HTML
   - Лучше определяет назначение элементов

2. **`html`** - анализ по HTML и тексту (устаревший)
   - Используется, если нужен анализ структуры HTML
   - Менее точный для семантического понимания

Выбор режима:
```bash
# Режим скриншота (по умолчанию)
ANALYSIS_MODE=screenshot node scripts/analyze-page-with-llm.js home

# Режим HTML (устаревший)
ANALYSIS_MODE=html node scripts/analyze-page-with-llm.js home
```

## Параметры

- `slug` - slug страницы для анализа (по умолчанию: `home`)
  - Примеры: `home`, `about_mgts`, `corporate_documents`

## Результаты

Скрипт создает следующие файлы в `mgts-backend/temp/page-analysis-llm/`:

- `{slug}_spec.json` - структурированное ТЗ с описанием всех секций
- `{slug}_screenshot.png` - скриншот страницы для визуальной проверки

## Формат вывода

```json
{
  "page": {
    "slug": "home",
    "url": "https://business.mgts.ru",
    "analyzedAt": "2026-01-09T...",
    "llmProvider": "perplexity",
    "llmModel": "llama-3.1-sonar-large-128k-online"
  },
  "metadata": {
    "title": "Главная страница"
  },
  "sections": [
    {
      "sectionIndex": 1,
      "type": "hero",
      "description": "Описание назначения секции",
      "title": "Заголовок секции",
      "subtitle": "Подзаголовок",
      "content": "Основное содержание",
      "interactiveElements": [],
      "images": [],
      "links": []
    }
  ]
}
```

## Настройка модели LLM

Можно изменить модель через переменную окружения:

```bash
export LLM_MODEL="llama-3.1-sonar-small-128k-online"  # Быстрее, но менее точная
export LLM_MODEL="llama-3.1-sonar-large-128k-online"    # Медленнее, но более точная (по умолчанию)
export LLM_MODEL="sonar"                               # Базовая модель
export LLM_MODEL="sonar-pro"                           # Pro версия
```

## Преимущества использования LLM

1. **Семантическое понимание** - LLM понимает контекст и назначение контента
2. **Визуальный анализ** - анализ по скриншоту позволяет понять визуальную структуру и дизайн
3. **Гибкость** - адаптируется к разным структурам страниц
4. **Обобщение** - создает краткие описания сложных блоков
5. **Классификация** - автоматически определяет типы контента
6. **Без дубликатов** - понимает, что секции уникальны
7. **Точные формулировки** - извлекает точные тексты заголовков и описаний со страницы

## Ограничения

- Требуется интернет-соединение для доступа к Perplexity API
- API ключ должен быть валидным
- Большие страницы могут требовать больше времени на анализ
- Стоимость API запросов зависит от модели и объема контента
