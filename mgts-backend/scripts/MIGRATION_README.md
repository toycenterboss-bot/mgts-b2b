# Миграция контента страниц услуг

## Описание

Скрипты для автоматической миграции контента (тарифы, FAQ, формы) из старых HTML блоков в новые стандартизированные секции на всех страницах услуг.

## Использование

### Вариант 1: Полный запуск (рекомендуется)

Автоматически перезапускает сервисы и запускает миграцию:

```bash
cd mgts-backend
./scripts/run-migration.sh
```

### Вариант 2: Ручной запуск

1. Перезапуск сервисов:
```bash
cd mgts-backend
./scripts/restart-services.sh
```

2. Запуск миграции:
```bash
cd mgts-backend
node scripts/migrate-all-service-pages.js
```

### Вариант 3: Одна страница

Для тестирования на одной странице:

```bash
cd mgts-backend
node scripts/migrate-service-page-content.js business/internet/gpon
```

## Что делает скрипт

1. **Извлечение тарифов** из старых блоков:
   - Находит блоки с классом `.card` или `div` с `h3` и ценой
   - Извлекает название, цену и характеристики
   - Переносит в секцию `service-tariffs`

2. **Извлечение FAQ** из старых блоков:
   - Находит блоки с вопросами и ответами
   - Извлекает пары вопрос-ответ
   - Переносит в секцию `service-faq`

3. **Удаление старых форм**:
   - Находит и удаляет старые формы заказа
   - Оставляет только новую секцию `service-order`

4. **Комментирование старых блоков**:
   - Старые блоки тарифов и FAQ закомментированы
   - Сохраняются для истории и возможного восстановления

## Логирование

Скрипт выводит подробную информацию:
- Прогресс обработки каждой страницы
- Время выполнения каждой страницы
- Найденные повторяющиеся структуры
- Извлеченные данные
- Ошибки (если есть)

## Обработка ошибок

- Ошибки на одной странице не останавливают обработку остальных
- Все ошибки логируются и отображаются в итоговом отчете
- HTTP запросы с таймаутом не зависают

## Требования

- Node.js 18+ (или установленный `node-fetch@2` для старых версий)
- Strapi запущен на `http://localhost:1337`
- Web-server на порту 8001 (опционально, для проверки)
