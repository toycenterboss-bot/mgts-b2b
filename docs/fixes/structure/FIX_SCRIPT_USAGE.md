# Скрипт автоматического исправления структуры страниц

## 📋 Описание

Скрипт `mgts-backend/scripts/fix-all-pages-structure.js` автоматически исправляет структуру HTML контента всех страниц в Strapi согласно правилам типизации из `CMS_CONTENT_TYPES.md`.

## 🔧 Что исправляет скрипт

1. **Добавляет класс `section-title` к заголовкам h2**
   - Все заголовки `h2` получают класс `section-title`

2. **Оборачивает контент в секции**
   - Контейнеры с контентом оборачиваются в `<section class="section">`

3. **Добавляет сетку для множественных карточек**
   - Если в контейнере несколько карточек, они оборачиваются в `<div class="grid grid-cols-N">`

4. **Исправляет `.grid-item` без `.grid`**
   - Элементы `.grid-item` оборачиваются в `.grid`

## 🚀 Использование

### Предварительный просмотр (рекомендуется сначала)

```bash
cd mgts-backend
node scripts/fix-all-pages-structure.js --dry-run
```

Это покажет, что будет исправлено, но не применит изменения.

### Применение исправлений с резервной копией

```bash
cd mgts-backend
node scripts/fix-all-pages-structure.js --backup
```

Создаст резервную копию всех страниц перед исправлением.

### Применение исправлений без резервной копии

```bash
cd mgts-backend
node scripts/fix-all-pages-structure.js
```

**⚠️ Внимание:** Изменения будут применены напрямую к Strapi!

## ⚠️ Важные замечания

1. **Всегда запускайте с `--dry-run` сначала** для проверки
2. **Используйте `--backup`** для создания резервной копии
3. **Убедитесь, что Strapi запущен** на `http://localhost:1337`
4. **Проверьте API токен** в `CONTEXT.md` если возникают ошибки авторизации

## 🔄 Восстановление из резервной копии

Если что-то пошло не так, можно восстановить страницы из резервной копии:

```bash
# Резервные копии находятся в:
strapi-backups/structure-fix-YYYY-MM-DDTHH-MM-SS/pages-backup.json
```

Используйте скрипт `restore-strapi-pages.js` для восстановления.

