# Исправление ошибки в setup_step1.ps1

## Проблема

При запуске скрипта `setup_step1.ps1` возникала ошибка:
```
В строке отсутствует завершающий символ: '.
Отсутствует закрывающий знак "}" в блоке операторов
```

## Причина

1. **Проблема с кодировкой** - русские символы и эмодзи могли быть неправильно закодированы
2. **Использование $PSScriptRoot** - может быть не определен в некоторых контекстах
3. **Эмодзи в строках** - могут вызывать проблемы с парсингом

## Исправления

### 1. Убраны эмодзи
- Заменены на текстовые сообщения
- Улучшена совместимость с разными версиями PowerShell

### 2. Исправлено определение родительской директории
**Было:**
```powershell
$parentDir = Split-Path -Parent $PSScriptRoot
```

**Стало:**
```powershell
$currentDir = Get-Location
$parentDir = Split-Path -Parent $currentDir.Path
```

### 3. Улучшена обработка ошибок
- Добавлены try-catch блоки
- Добавлена проверка кодов возврата
- Восстановление текущей директории при ошибках

### 4. Улучшена проверка существования файлов
```powershell
if (Test-Path ".\check_environment.ps1") {
    & .\check_environment.ps1
}
```

## Использование

Теперь скрипт можно запускать без ошибок:

```powershell
.\setup_step1.ps1
```

## Альтернатива (если скрипт все еще не работает)

Выполните команды вручную:

```powershell
# 1. Проверка окружения
.\check_environment.ps1

# 2. Переход в родительскую директорию
cd ..

# 3. Создание директории
mkdir mgts-cms
cd mgts-cms

# 4. Установка Strapi CLI
npm install -g @strapi/strapi

# 5. Создание проекта
npx create-strapi-app@latest mgts-backend --quickstart
```

## Проверка исправления

Запустите скрипт снова:
```powershell
.\setup_step1.ps1
```

Если ошибка повторяется, используйте альтернативный способ (команды вручную).
