# Исправление проблемы с компонентами на страницах третьего уровня

## Проблема
На страницах третьего уровня (например, `business/internet/office/index.html`) меню и footer не загружались.

## Причины

1. **Неправильные пути к скрипту** - на страницах третьего уровня был путь `../../js/components-loader.js` вместо `../../../js/components-loader.js`
2. **Остатки старого кода** - на некоторых страницах остались блоки старого mega-menu, которые мешали загрузке компонентов
3. **Неправильное вычисление базового пути** - функция `getBasePath()` могла неправильно вычислять путь для третьего уровня

## Выполненные исправления

### 1. Исправлены пути к скриптам
- ✅ Обновлено **24 страницы** третьего уровня
- ✅ Путь изменен с `../../js/components-loader.js` на `../../../js/components-loader.js`

### 2. Удалены остатки старого кода
- ✅ Удалены старые блоки mega-menu с **24 страниц**
- ✅ Очищены дублирующиеся блоки между header и breadcrumbs

### 3. Улучшено вычисление базового пути
- ✅ Добавлена отладочная информация в консоль
- ✅ Улучшена логика вычисления для HTTP протокола
- ✅ Правильный расчет для всех уровней вложенности

## Проверка

Для страницы `business/internet/office/index.html`:
- Путь к скрипту: `../../../js/components-loader.js` ✅
- Базовый путь для компонентов: `../../../` ✅
- Путь к компонентам: `../../../components/header.html` ✅

## Результат

Теперь на всех страницах третьего уровня:
- ✅ Header загружается корректно
- ✅ Footer загружается корректно
- ✅ Mega-menu работает
- ✅ Все пути правильные

- Сообщения о загрузке компонентов
- Вычисленный базовый путь
- Путь к компонентам
- Ошибки загрузки
