# Исправление отображения страниц раздела "О компании"

## Проблема

Страницы раздела "О компании" не отображали контент из Strapi, хотя структура HTML в Strapi была корректной.

## Причина

1. **Класс `service-card` не был определен в типизации** - класс использовался на страницах раздела "О компании", но не был описан в `CMS_CONTENT_TYPES.md`.

2. **Элементы `service-card` не обрабатывались после вставки контента** - функция `enhanceTariffCards` вызывалась только для контейнеров с `.grid`, но на страницах раздела "О компании" элементы `service-card` находятся прямо внутри `.container` без обертки в `.grid`.

3. **Отсутствие обработки после замены placeholder-секций** - когда контент заменял placeholder-секции, элементы `service-card` не обрабатывались функцией `enhanceServiceCard`.

## Решение

### 1. Добавлено определение класса `service-card` в `CMS_CONTENT_TYPES.md`

Добавлено полное описание класса `.service-card`:
- Назначение: специальная карточка для отображения услуг, информации или навигации
- Использование: для услуг, информационных блоков, навигационных карточек
- Структура: может содержать заголовки, параграфы, списки, ссылки, иконки
- Модификаторы: `data-card-type="navigation"`, `data-card-type="info"`, `data-card-type="service"`
- Обработка: автоматически преобразуется в структуру с `.service-card-body` и `.service-card-icon`

### 2. Добавлена обработка `service-card` после замены контента

В `cms-loader.js` добавлены вызовы `enhanceTariffCards` после:
- Замены placeholder-секций (строка ~1517)
- Замены существующих секций (строка ~1470)
- Замены секций, найденных на странице (строка ~1607)

Это гарантирует, что все элементы `service-card` будут правильно обработаны независимо от того, как контент был вставлен.

## Проверка

Для проверки работы исправлений можно использовать скрипт:
```bash
cd mgts-backend
node scripts/debug-about-pages.js
```

Скрипт показывает:
- Структуру контента из Strapi
- Количество элементов `service-card`
- Детали каждого элемента
- Проверку обработки процессорами
