# Использование типов гридов через CMS

## Что было реализовано

✅ Добавлена поддержка явного задания типов карточек и гридов через Strapi CMS

## Типы карточек (cardType)

### `navigation` - Карточка-ссылка
- Вся карточка кликабельна (сама карточка - это `<a>` тег)
- Кнопка "Узнать больше" **НЕ создается**
- Используется для навигационных карточек, ведущих на другие страницы

### `info` - Информационная карточка
- Нет ссылок
- Кнопка "Узнать больше" **НЕ создается**
- Используется для информационных карточек без ссылок

### `service` - Карточка услуги
- Может иметь ссылку внутри карточки
- Кнопка "Узнать больше" создается **только если есть реальная ссылка** (не `#` и не пустая)
- Используется для карточек услуг

### `tariff` - Тарифная карточка
- С ценой, условиями, списком функций
- Кнопка "Заказать" создается всегда
- Используется для тарифных карточек

## Типы гридов (gridType)

### `navigation` - Грид с карточками-ссылками
- Все карточки в гриде обрабатываются как `navigation`
- Кнопки "Узнать больше" не создаются

### `info` - Грид с информационными карточками
- Все карточки в гриде обрабатываются как `info`
- Кнопки "Узнать больше" не создаются

### `service` - Грид с карточками услуг
- Все карточки в гриде обрабатываются как `service`
- Кнопки создаются только если есть ссылки

### `tariff` - Грид с тарифными карточками
- Все карточки в гриде обрабатываются как `tariff`
- Кнопки "Заказать" создаются всегда

### `mixed` - Смешанный грид (по умолчанию)
- Тип каждой карточки определяется индивидуально (по `cardType` или автоопределением)

## Как использовать в Strapi

### 1. Для отдельной карточки

1. Откройте страницу в Strapi Admin Panel
2. Найдите компонент `Card` в секции
3. В поле **Card Type** выберите нужный тип:
   - `navigation` - для карточек-ссылок
   - `info` - для информационных карточек
   - `service` - для карточек услуг
   - `tariff` - для тарифных карточек
4. Сохраните страницу

### 2. Для всего грида

1. Откройте страницу в Strapi Admin Panel
2. Найдите компонент `Section Grid` в секции
3. В поле **Grid Type** выберите нужный тип:
   - `navigation` - для грида с карточками-ссылками
   - `info` - для грида с информационными карточками
   - `service` - для грида с карточками услуг
   - `tariff` - для грида с тарифными карточками
   - `mixed` - для смешанного грида (по умолчанию)
4. Сохраните страницу

### 3. Приоритет определения типа

1. **`cardType` карточки** (если задан) - имеет наивысший приоритет
2. **`gridType` секции** (если задан и не `mixed`) - применяется ко всем карточкам в гриде
3. **Автоопределение** - используется если тип не задан явно

## Примеры использования

### Пример 1: Навигационные карточки

**В Strapi:**
- Section Grid → Grid Type: `navigation`
- Или для каждой карточки: Card → Card Type: `navigation`

**Результат:**
- Вся карточка становится ссылкой
- Кнопка "Узнать больше" не создается

### Пример 2: Информационные карточки

**В Strapi:**
- Section Grid → Grid Type: `info`
- Или для каждой карточки: Card → Card Type: `info`

**Результат:**
- Карточки отображаются как информационные
- Кнопка "Узнать больше" не создается

### Пример 3: Смешанный грид

**В Strapi:**
- Section Grid → Grid Type: `mixed` (или не задавать)
- Для каждой карточки задать свой `cardType`

**Результат:**
- Каждая карточка обрабатывается согласно своему типу

## Технические детали

### Data-атрибуты

При рендеринге контента из CMS, JavaScript автоматически:
- Читает `data-card-type` атрибут карточки (если есть)
- Читает `data-grid-type` атрибут грида (если есть)
- Использует эти атрибуты для определения типа

### Обратная совместимость

Если тип не задан в CMS:
- Используется автоопределение типа карточки (текущая логика)
- Работает как раньше, без изменений

## Рекомендации

1. **Для навигационных карточек** - используйте `cardType: navigation` или `gridType: navigation`
2. **Для информационных карточек** - используйте `cardType: info` или `gridType: info`
3. **Для карточек услуг** - используйте `cardType: service` или `gridType: service`
4. **Для тарифных карточек** - используйте `cardType: tariff` или `gridType: tariff`
5. **Для смешанных гридов** - задавайте `cardType` для каждой карточки отдельно
