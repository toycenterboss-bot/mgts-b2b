# Правила расположения элементов верхнего уровня

## Обзор

Данный документ определяет правила расположения HTML элементов, которые находятся внутри `.container`, но вне блоков `div` или `section`.

## Результаты анализа

На основе анализа 42 страниц проекта:

- **`<h2>`**: 86 использований на всех 42 страницах
  - Все внутри `.container` ✅
  - Все внутри секций ✅
  - Используются как заголовки секций ✅

- **`<p>`**: 25 использований на 23 страницах
  - Все внутри `.container` ✅
  - Все внутри секций ✅
  - Используются для описаний и вводного текста ✅

## Разрешенные элементы верхнего уровня

### 1. Заголовки (`<h1>`, `<h2>`, `<h3>`, `<h4>`, `<h5>`, `<h6>`)

**Правила:**
- ✅ Могут находиться внутри `.container` напрямую
- ✅ Должны иметь класс `section-title` если это заголовок секции
- ✅ Должны быть первым элементом в секции или перед контентом секции
- ✅ Должны быть внутри секции (не между секциями)
- ❌ Не должны находиться между секциями (вне container)
- ❌ Не должны находиться после всех секций

**Правильная структура:**
```html
<section class="section">
    <div class="container">
        <h2 class="section-title">Заголовок секции</h2>
        <p>Описание секции</p>
        <!-- остальной контент -->
    </div>
</section>
```

**Неправильная структура:**
```html
<!-- ❌ НЕПРАВИЛЬНО: h2 между секциями -->
<section class="section">...</section>
<h2>Заголовок</h2>
<section class="section">...</section>

<!-- ❌ НЕПРАВИЛЬНО: h2 без класса section-title -->
<section class="section">
    <div class="container">
        <h2>Заголовок</h2>
    </div>
</section>
```

### 2. Параграфы (`<p>`)

**Правила:**
- ✅ Могут находиться внутри `.container` напрямую
- ✅ Должны быть внутри секции (внутри `.container` секции)
- ✅ Могут использоваться для краткого описания перед основным контентом
- ❌ Не должны находиться между секциями
- ❌ Не должны находиться после всех секций

**Правильная структура:**
```html
<section class="section">
    <div class="container">
        <h2 class="section-title">Заголовок</h2>
        <p>Краткое описание секции</p>
        <!-- остальной контент -->
    </div>
</section>
```

### 3. Списки (`<ul>`, `<ol>`, `<li>`)

**Правила:**
- ✅ Могут находиться внутри `.container` напрямую
- ✅ Должны быть внутри секции
- ✅ Могут быть частью контента секции

**Правильная структура:**
```html
<section class="section">
    <div class="container">
        <h2 class="section-title">Список</h2>
        <ul>
            <li>Элемент 1</li>
            <li>Элемент 2</li>
        </ul>
    </div>
</section>
```

### 4. Инлайн элементы (`<span>`, `<strong>`, `<em>`, `<a>`)

**Правила:**
- ✅ Могут находиться внутри `.container` как часть текстового контента
- ✅ Должны быть внутри секции
- ✅ Должны быть частью текста (внутри `<p>`, `<h2>` и т.д.)
- ❌ Не должны быть отдельными элементами верхнего уровня

**Правильная структура:**
```html
<section class="section">
    <div class="container">
        <p>Текст с <strong>выделением</strong> и <a href="#">ссылкой</a></p>
    </div>
</section>
```

## Запрещенные элементы верхнего уровня

### ❌ Блочные элементы вне секций:

- `<div>` - должен быть внутри секции или быть частью структуры (`.container`, `.grid`, `.card` и т.д.)
- `<section>` - должен быть основным контейнером
- `<article>`, `<aside>`, `<nav>` - должны быть внутри секций

### ❌ Интерактивные элементы вне контекста:

- `<form>` - должен быть внутри секции (обычно `service-order`)
- `<button>` - должен быть внутри формы или карточки
- `<input>`, `<textarea>`, `<select>` - должны быть внутри формы

### ❌ Медиа элементы вне контекста:

- `<img>` - должен быть внутри карточки, секции или другого контейнера
- `<video>`, `<audio>` - должны быть внутри секции

## Правила валидации

### 1. Все элементы верхнего уровня должны быть внутри секции

```html
<!-- ✅ ПРАВИЛЬНО -->
<section class="section">
    <div class="container">
        <h2 class="section-title">Заголовок</h2>
        <p>Текст</p>
    </div>
</section>
```

### 2. Элементы не должны находиться между секциями

```html
<!-- ❌ НЕПРАВИЛЬНО -->
<section class="section">...</section>
<h2>Заголовок</h2>
<section class="section">...</section>

<!-- ✅ ИСПРАВЛЕНО -->
<section class="section">...</section>
<section class="section">
    <div class="container">
        <h2 class="section-title">Заголовок</h2>
    </div>
</section>
```

### 3. Элементы не должны находиться после всех секций

```html
<!-- ❌ НЕПРАВИЛЬНО -->
<section class="section">...</section>
<p>Текст</p>

<!-- ✅ ИСПРАВЛЕНО -->
<section class="section">...</section>
<section class="section">
    <div class="container">
        <p>Текст</p>
    </div>
</section>
```

### 4. Заголовки должны иметь класс `section-title`

```html
<!-- ✅ ПРАВИЛЬНО -->
<h2 class="section-title">Заголовок секции</h2>

<!-- ❌ НЕПРАВИЛЬНО -->
<h2>Заголовок секции</h2>
```

**Исключения:**
- Заголовки в формах заказа (`service-order`) могут не иметь `section-title`
- Заголовки с специальными классами (`.order-form__title`, `.faq-question` и т.д.) не требуют `section-title`
- Заголовки внутри карточек (`.card-title`, `.tariff-card__title`) не требуют `section-title`

## Автоматическая проверка и исправление

Используйте скрипт для проверки элементов верхнего уровня:
```bash
# Проверка (без изменений)
node mgts-backend/scripts/check-top-level-elements.js --dry-run

# Исправление
node mgts-backend/scripts/check-top-level-elements.js
```

Скрипт автоматически:
- Обернет элементы между секциями в новую секцию
- Обернет элементы после всех секций в новую секцию
- Добавит класс `section-title` к заголовкам без этого класса

## Резюме правил

### Обязательные правила:
1. Все элементы верхнего уровня должны быть внутри секции
2. Элементы не должны находиться между секциями
3. Элементы не должны находиться после всех секций
4. Заголовки секций должны иметь класс `section-title`

### Разрешенные элементы:
- Заголовки (`<h1>` - `<h6>`) - внутри секции, с классом `section-title`
- Параграфы (`<p>`) - внутри секции
- Списки (`<ul>`, `<ol>`, `<li>`) - внутри секции
- Инлайн элементы (`<span>`, `<strong>`, `<em>`, `<a>`) - внутри текста

### Запрещенные элементы:
- Блочные элементы (`<div>`, `<section>`) - вне структуры
- Интерактивные элементы (`<form>`, `<button>`) - вне контекста
- Медиа элементы (`<img>`, `<video>`) - вне контейнеров
