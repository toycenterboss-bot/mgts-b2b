## План действий

### Шаг 1: Подключение cms-processors.js

**Проблема:** `cms-processors.js` не подключен в HTML файлах

**Решение:**
1. Добавить `<script src="js/cms-processors.js"></script>` перед `cms-loader.js` во всех HTML файлах
2. Проверить загрузку модуля

**Файлы для изменения:** Все 42+ HTML файла

### Шаг 2: Рефакторинг cms-loader.js

**Текущее состояние:**
- `renderContent()` - ~2500 строк сложной логики
- Смешивает разные типы обработки
- Трудно поддерживать и изменять

**Целевое состояние:**
- Использовать `ProcessorManager` для обработки
- Упростить код до ~500-800 строк
- Изолировать изменения в процессорах

**Подход:**
1. Создать функцию-обертку `processWithManager()`
2. Постепенно заменить логику в `renderContent()`
3. Сохранить все специальные случаи (главная страница, service pages)

### Шаг 3: Тестирование

**Страницы для тестирования:**
- Service pages (business/internet/gpon)
- Category pages (business/telephony)
- Regular pages (about, government)
- Special pages (main_page, contacts)

## Преимущества после интеграции

1. **Упрощение кода:** С ~2500 до ~500-800 строк
2. **Модульность:** Каждый тип контента - отдельный процессор
3. **Изоляция:** Изменения в одном процессоре не влияют на другие
4. **Расширяемость:** Легко добавлять новые типы контента
5. **Тестируемость:** Каждый процессор можно тестировать отдельно
