# Документация проекта МГТС

Эта папка содержит всю документацию проекта, организованную по тематическим разделам.

## Структура документации

### 📚 CMS (Content Management System)
- **`cms/content-types/`** - Типизация контента, правила структуры HTML
- **`cms/integration/`** - Интеграция Strapi CMS с фронтендом
- **`cms/grid-types/`** - Типы grid контейнеров и их использование
- **`cms/structure/`** - Правила структуры контента, иерархия классов

### 🔧 Исправления (Fixes)
- **`fixes/structure/`** - Документация по исправлениям структуры HTML
- **`fixes/pages/`** - Исправления отдельных страниц
- **`fixes/styles/`** - Исправления CSS стилей

### 📦 Установка и настройка
- **`installation/setup/`** - Инструкции по установке и настройке окружения
- **`installation/migration/`** - Документация по миграции данных
- **`installation/troubleshooting/`** - Решение типичных проблем

### 📊 Анализ и отчеты
- **`analysis/reports/`** - JSON файлы с результатами различных анализов

### 📖 Руководства
- **`guides/editor/`** - Руководство для редакторов контента
- **`guides/development/`** - Документация для разработчиков
- **`guides/usage/`** - Инструкции по использованию системы

### 📈 Статусы и прогресс
- **`status/progress/`** - Документация о текущем прогрессе проекта
- **`status/completed/`** - Завершенные задачи и этапы

### 🎯 Проект
- **`project/`** - Основная документация проекта (CONTEXT.md, структура сайта)
- **`setup/`** - Настройка внешних сервисов (Cloudinary, Local Storage)

## Основные файлы

- **`CONTEXT.md`** (в `project/`) - Основной контекст проекта, API токены
- **`CMS_CONTENT_TYPES.md`** (в `cms/content-types/`) - Полная типизация контента
- **`FILE_ORGANIZATION_PLAN.md`** - План организации файлов

## Скрипты организации

- **`analyze-files.js`** - Анализ и категоризация файлов
- **`organize-files.js`** - Автоматическая организация файлов по папкам

## Быстрый старт

1. Для просмотра правил контента: `cms/content-types/CMS_CONTENT_TYPES.md`
2. Для установки: `installation/setup/`
3. Для решения проблем: `installation/troubleshooting/`
4. Для разработки: `guides/development/`
