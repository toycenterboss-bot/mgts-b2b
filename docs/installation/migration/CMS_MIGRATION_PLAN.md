### 4.2. Рефакторинг cms-loader.js

Заменить текущую логику обработки на использование `ProcessorManager`:

```javascript
// Вместо текущей логики:
const processorManager = new CMSProcessors.ProcessorManager();

// Разделить секции
const { specialSections, regularSections } = processorManager.separateSections(sections);

// Обработать специальные секции
specialSections.forEach(section => {
    const processed = processorManager.process(section, context);
    if (processed) {
        // Вставить в DOM
    }
});

// Обработать обычные секции
regularSections.forEach(section => {
    const processed = processorManager.process(section, context);
    if (processed) {
        // Сопоставить с существующими секциями
    }
});
```

## Преимущества модульной системы

1. **Изоляция:** Изменения в одном процессоре не влияют на другие
2. **Расширяемость:** Легко добавить новый тип контента
3. **Тестируемость:** Каждый процессор можно тестировать отдельно
4. **Поддерживаемость:** Код проще понимать и изменять
5. **Типизация:** Четкое определение структуры каждого типа контента
