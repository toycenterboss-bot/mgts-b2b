# Проверка установки Strapi

## Быстрая проверка

Запустите скрипт проверки:

```powershell
cd C:\Users\abefremov\SiteMGTS
.\check_strapi_setup.ps1
```

## Ручная проверка

### 1. Проверка структуры проекта

```powershell
cd C:\mgts-cms\mgts-backend
dir
```

Должны быть следующие директории и файлы:
- ✅ `package.json`
- ✅ `node_modules/`
- ✅ `config/`
- ✅ `src/`
- ✅ `.env` (может отсутствовать, создастся при первом запуске)

### 2. Проверка установленных зависимостей

```powershell
npm list @strapi/strapi
```

Должна отобразиться версия Strapi, например:
```
mgts-backend@1.0.0
└── @strapi/strapi@4.15.0
```

### 3. Проверка конфигурации

```powershell
# Проверить database.js
type config\database.js

# Проверить server.js
type config\server.js
```

### 4. Проверка запуска

```powershell
npm run develop
```

**Ожидаемый результат:**
- Сервер запускается на `http://localhost:1337`
- В консоли нет критических ошибок
- Открывается браузер с админ-панелью

### 5. Проверка админ-панели

1. Откройте браузер: `http://localhost:1337/admin`
2. Должна открыться страница регистрации администратора
3. Если страница открывается - установка успешна!

## Типичные проблемы

### Проблема: `node_modules` не найдены

**Решение:**
```powershell
cd C:\mgts-cms\mgts-backend
npm install
```

### Проблема: Порт 1337 занят

**Решение:**
1. Остановите другой процесс на порту 1337
2. Или измените порт в `.env`:
   ```
   PORT=1338
   ```

### Проблема: Ошибка при запуске `npm run develop`

**Решение:**
1. Проверьте логи ошибок
2. Убедитесь, что все зависимости установлены: `npm install`
3. Проверьте версию Node.js: `node --version` (должна быть 18+)

### Проблема: Админ-панель не открывается

**Решение:**
1. Проверьте, что сервер запущен
2. Проверьте URL: `http://localhost:1337/admin` (не `http://localhost:1337`)
3. Проверьте консоль браузера на ошибки
4. Проверьте файрвол и антивирус
