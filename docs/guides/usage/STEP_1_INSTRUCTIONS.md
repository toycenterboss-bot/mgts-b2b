# Инструкции по выполнению Этапа 1

## Текущий статус

❌ **Node.js не установлен** - требуется установка
❌ **npm не установлен** - устанавливается вместе с Node.js
❌ **Git не установлен** - рекомендуется установить

## Пошаговая инструкция

### Шаг 1: Установка Node.js и npm

1. **Откройте браузер** и перейдите на https://nodejs.org/
2. **Скачайте LTS версию** (рекомендуется 18.x или выше)
   - Нажмите кнопку "Download Node.js (LTS)"
   - Выберите версию для Windows (64-bit)
3. **Запустите установщик**
   - Дважды кликните на скачанный файл
4. **Следуйте инструкциям установщика:**
   - Нажмите "Next" на всех экранах
   - **Важно:** Убедитесь, что опция "Add to PATH" отмечена (обычно включена по умолчанию)
   - Нажмите "Install"
   - Дождитесь завершения установки
   - Нажмите "Finish"
5. **Перезапустите PowerShell/терминал**
   - Закройте текущее окно PowerShell
   - Откройте новое окно PowerShell
6. **Проверьте установку:**
   ```powershell
   node --version
   npm --version
   ```
   Должны отобразиться версии без ошибок.

### Шаг 2: Установка Git (рекомендуется)

1. **Откройте браузер** и перейдите на https://git-scm.com/download/win
2. **Скачайте установщик**
   - Нажмите на ссылку для скачивания
3. **Запустите установщик**
   - Дважды кликните на скачанный файл
4. **Следуйте инструкциям:**
   - Можно оставить все настройки по умолчанию
   - Нажимайте "Next" до завершения
   - Нажмите "Install"
5. **Перезапустите PowerShell**
6. **Проверьте установку:**
   ```powershell
   git --version
   ```

### Шаг 3: Проверка окружения

После установки всех инструментов:

1. **Перейдите в директорию проекта:**
   ```powershell
   # Если скрипты находятся в C:\runs, скопируйте их в SiteMGTS или используйте полный путь
   cd C:\Users\abefremov\SiteMGTS
   ```

2. **Запустите скрипт проверки:**
   ```powershell
   # Если скрипт в другой директории, используйте полный путь:
   # C:\runs\check_environment.ps1
   .\check_environment.ps1
   ```

3. **Если все проверки пройдены**, запустите скрипт настройки:
   ```powershell
   # Важно: setup_step1.ps1 должен быть запущен из директории SiteMGTS
   # или из C:\runs, но тогда он должен знать путь к SiteMGTS
   .\setup_step1.ps1
   ```
   
   **Примечание:** Если скрипт запускается из `C:\runs`, он автоматически определит путь к проекту.

### Шаг 4: Создание проекта Strapi

После успешной установки всех инструментов:

1. **Перейдите в родительскую директорию:**
   ```powershell
   cd ..
   ```

2. **Создайте директорию для CMS:**
   ```powershell
   mkdir mgts-cms
   cd mgts-cms
   ```

3. **Установите Strapi CLI глобально:**
   ```powershell
   npm install -g @strapi/strapi
   ```
   Это может занять несколько минут.

4. **Создайте новый проект Strapi:**
   ```powershell
   npx create-strapi-app@latest mgts-backend --quickstart
   ```
   
   **Параметры:**
   - Database: SQLite (для разработки) - выберите этот вариант
   - Template: Quickstart
   
   Это займет 5-10 минут.

5. **После создания проекта:**
   ```powershell
   cd mgts-backend
   npm run develop
   ```

6. **Откройте браузер:**
   - Перейдите на http://localhost:1337/admin
   - Создайте первого администратора:
     - First name: Admin
     - Last name: User
     - Email: admin@mgts.ru
     - Password: [надежный пароль - сохраните его!]

## Решение проблем

### Проблема: "node не является внутренней или внешней командой"

**Решение:**
1. Перезапустите PowerShell/терминал
2. Если не помогло, проверьте переменную PATH:
   ```powershell
   $env:PATH -split ';' | Select-String node
   ```
3. Если Node.js не в PATH, добавьте вручную или переустановите Node.js

### Проблема: "npm не найден"

**Решение:**
- npm устанавливается вместе с Node.js
- Переустановите Node.js, убедившись, что опция "Add to PATH" отмечена

### Проблема: Ошибки при установке пакетов

**Решение:**
```powershell
# Очистить кэш npm
npm cache clean --force

# Попробовать снова
npm install -g @strapi/strapi
```
