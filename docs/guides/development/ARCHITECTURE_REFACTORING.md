# Рефакторинг архитектуры сайта

## Цель
Упростить поддержку сайта, вынеся общие компоненты (header, footer, sidebar) в отдельные файлы и используя их на всех страницах.

## Структура компонентов

### Созданные компоненты:
1. **`components/header.html`** - Общий header с навигацией и mega-menu
2. **`components/footer.html`** - Общий footer со ссылками
3. **`components/sidebar-about.html`** - Боковое меню для раздела "О компании"

### JavaScript модули:
1. **`js/components-loader.js`** - Загрузка компонентов через fetch API
2. **`js/content-loader.js`** - Загрузка контента через iframe или fetch API

## Использование компонентов

### Вариант 1: JavaScript загрузка (рекомендуется)

На странице размещаем плейсхолдеры:
```html
<div data-component="header"></div>
<div data-component="footer"></div>
<div data-component="sidebar-about"></div>
```

Подключаем скрипт:
```html
<script src="js/components-loader.js"></script>
```

Компоненты автоматически загрузятся при загрузке страницы.

### Вариант 2: Iframe загрузка

Для использования iframe:
```html
<iframe src="components/header.html" style="border: none; width: 100%;"></iframe>
```

**Недостатки iframe:**
- Плохо для SEO
- Проблемы с навигацией
- Сложнее стилизация
- Медленнее загрузка

## Обновление путей

Компоненты автоматически обновляют пути в зависимости от уровня вложенности страницы:
- Корень: `index.html`
- Уровень 1: `../index.html`
- Уровень 2: `../../index.html`
- И т.д.

## Следующие шаги

1. Обновить все страницы для использования компонентов
2. Создать отдельные файлы контента для загрузки через iframe (если требуется)
3. Протестировать навигацию и загрузку компонентов
4. Обновить пути в компонентах при необходимости
