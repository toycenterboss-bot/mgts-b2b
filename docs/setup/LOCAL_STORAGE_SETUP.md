**Функциональность:**
- Загружает изображения напрямую в Strapi Media Library
- Использует локальное хранилище
- Сохраняет результаты миграции

---

## 🚀 Как это работает

### Локальное хранилище Strapi

1. **Загрузка файлов:**
   - Файлы сохраняются в `mgts-backend/public/uploads/`
   - Strapi автоматически создает структуру папок по датам
   - URL файлов: `http://localhost:1337/uploads/...`

2. **Доступ к файлам:**
   - Файлы доступны через публичный URL
   - Не требуется внешних сервисов
   - Полностью автономное решение

3. **Структура:**
   ```
   mgts-backend/
     public/
       uploads/
         2024/
           12/
             filename.jpg
   ```

## 📝 Использование

### Загрузить изображение через админ-панель:

1. Откройте http://localhost:1337/admin
2. Перейдите в **Media Library**
3. Нажмите **Add new assets**
4. Выберите файл и загрузите
5. Файл сохранится локально в `public/uploads/`

### Мигрировать существующие изображения:

1. **Создать API токен:**
   - Settings → API Tokens → Create new API Token
   - Name: `Image Upload Script`
   - Token type: `Full access`

2. **Запустить скрипт:**
   ```bash
   cd /Users/andrey_efremov/Downloads/runs/mgts-backend
   export STRAPI_API_TOKEN="your_token_here"
   node scripts/migration/upload-images-local.js
   ```

3. **Проверить результат:**
   - Media Library → должны появиться загруженные изображения
   - URL изображений будут вида: `http://localhost:1337/uploads/...`

```typescript
  upload: {
    config: {

      providerOptions: {
        // Можно настроить размеры, форматы и т.д.
      },
      actionOptions: {
        upload: {},
        uploadStream: {},
        delete: {},
      },
    },
  },
});
```

### Ограничения:

- **Размер файла:** Настраивается в Strapi (по умолчанию 200MB)

- **Хранилище:** Зависит от размера диска сервера
