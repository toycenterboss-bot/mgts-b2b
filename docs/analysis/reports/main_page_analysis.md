# Анализ и рекомендации по улучшению главной страницы

## 📋 Текущие проблемы

### 1. **Семантика HTML**
- ❌ Использование множества `<div>` вместо семантических тегов (`<section>`, `<article>`, `<header>`)
- ❌ Отсутствие структурированных заголовков (h1-h6)
- ❌ Нет семантической разметки для навигации и контента

### 2. **Inline стили**
- ❌ `style="height: 364px;"` и `style="height: 0px;"` в HTML
- ❌ Стили должны быть в CSS файлах

### 3. **SVG код**
- ❌ Встроенные SVG напрямую в HTML (увеличивает размер файла)
- ❌ Дублирование SVG кода
- ✅ Рекомендация: использовать спрайты или отдельные файлы

### 4. **Структура секций**
- ❌ Непоследовательная структура секций
- ❌ Разные подходы к оберткам контента
- ❌ Отсутствие единообразия в классах

### 5. **Доступность (A11y)**
- ❌ Отсутствие `alt` атрибутов для изображений
- ❌ Нет `aria-label` для интерактивных элементов
- ❌ Отсутствие `role` атрибутов где необходимо

### 6. **SEO**
- ❌ Отсутствие структурированных данных
- ❌ Неоптимальная иерархия заголовков

## ✅ Рекомендации по улучшению

### 1. **Семантическая структура**
- Использовать `<section>` для каждой секции
- Использовать `<header>` для заголовков секций
- Использовать `<article>` для независимых блоков контента
- Правильная иерархия заголовков (h2 для секций, h3 для подсекций)

### 2. **Унификация классов**
- Использовать классы из дизайн-системы проекта
- Привести все секции к единому формату: `section section-{name}`
- Использовать `container` и `container-mgts` последовательно

### 3. **Карточки**
- Унифицировать структуру карточек
- Использовать классы `card`, `service-card` из дизайн-системы
- Добавить hover эффекты и переходы

### 4. **Оптимизация**
- Убрать все inline стили
- Вынести SVG в отдельные файлы или использовать спрайты
- Минимизировать вложенность div

### 5. **Доступность**
- Добавить `alt` атрибуты для всех изображений
- Добавить `aria-label` для интерактивных элементов
- Использовать семантические теги для улучшения доступности

### 6. **Адаптивность**
- Убедиться, что все секции корректно отображаются на мобильных
- Использовать responsive классы из дизайн-системы
