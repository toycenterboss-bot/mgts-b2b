# MGTS CMS-loader (клиентский JS для интерактива)

Цель: единый модульный loader для интерактивных компонентов в HTML‑блоках Stitch и в CMS.

## Принципы
- **Канонические обработки**: глобальные паттерны (header/mega‑menu, tabs, dropdown, modal и т.п.)
- **Обработки внутри контента**: специфичные для конкретных блоков/секций.
- **Overrides по страницам**: один и тот же тип компонента может иметь разную логику на разных страницах.

## Как это работает
`cms-loader.js`:
- определяет `window.MGTS_CMS_LOADER`
- собирает контекст страницы (например, `data-page`)
- запускает обработчики по приоритету
- предотвращает двойную инициализацию (per-module marker)

## Подключение
Для теста в локальных страницах:
- подключить скрипт `design/cms_loader/cms-loader.js` (через `build_html_pages.py`)

Для CMS:
- скопировать файл в публичные ассеты и подключить на стороне фронта/темы CMS

## CMS Adapter (пример интеграции)
Файл: `design/cms_loader/cms-adapter-example.js`

Что показывает:
- как слушать `mgts:open` (cancelable) и выбирать “модалка vs отдельный экран”
- как подставлять контент в модалку (демо-логика вместо реального API)

### Проверка с реальным Strapi (news)
Адаптер умеет (best-effort) подтягивать реальный контент новости из Strapi, если:
- `data-content-type="news"`
- `data-content-id="<slug>"` (slug новости в Strapi)

По умолчанию ожидается Strapi на `http://localhost:1337`.
Можно переопределить через query:
- `?strapi=http://localhost:1337`

## Контекст страницы
`<body data-page="tpl_home">` (выставляет сборщик)

## Регистрация override
В `cms-loader.js` можно добавить обработчик с:
- `pages: ["tpl_home", "tpl_news_list"]`
- `priority: 100` (выше = раньше)
- `selector` чтобы применяться только к нужным блокам
